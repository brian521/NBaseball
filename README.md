# ⚾ Number Baseball Game
 
> 시크릿 숫자 3개를 플레이어들이 번갈아가며 맞춰보는 멀티플레이어 숫자 야구 게임
 
---
 
## 게임 소개
 
**Number Baseball**은 고전 숫자 야구 게임을 멀티플레이어로 즐길 수 있는 실시간 웹 게임입니다.  
서로 다른 숫자 3개로 이루어진 시크릿 숫자를 가장 먼저 맞히는 플레이어가 승리합니다.

## UI 구성
 
| 영역 | 설명 |
|------|------|
| **좌측** | 실시간 채팅 (추측 기록 및 결과 포함) |
| **중앙 상단** | 알림 메시지 (게임 이벤트, 턴 변경 등) |
| **중앙** | 현재 턴의 남은 시간 타이머 |
| **우측** | 전체 플레이어 목록 (현재 턴인 플레이어는 🔴 빨간 이름으로 표시) |
| **하단** | 숫자 입력 채팅 칸 (자신의 턴에만 활성화) |

## 게임 규칙
 
### 기본 규칙
- 플레이어들은 **번갈아가며** 시크릿 숫자 3개를 추측합니다.
- 자신의 턴에 **채팅 입력 칸**에 서로 다른 3개의 숫자를 입력합니다.
 
### 판정 기준
 
| 결과 | 조건 |
|------|------|
| ⚾ **스트라이크** | 숫자와 자리 모두 일치 |
| 🔵 **볼** | 숫자는 있지만 자리가 다름 |
| ❌ **아웃** | 맞는 숫자가 하나도 없음 |
 
### 승패 조건
- **3 Strike** → 해당 플레이어 **승리**, 게임 종료
- 각 플레이어의 추측 기회는 **최대 3번**
- 턴 제한시간 **15초** 이내에 입력하지 못하면 기회 1회 소모
- 우승자 없이 모든 플레이어의 기회가 소진되면 **무승부**로 게임 종료
 
### 라운드 초기화
게임이 종료되면 (승리 또는 무승부):
1. 새로운 시크릿 숫자 생성
2. 모든 플레이어의 추측 기회 초기화
3. 다음 라운드 자동 시작

## 구현 과정
 
### 1. 프레임워크 설계
 
 
| 클래스 | 역할 |
|--------|------|
| `ANBGameModeBase` | 게임 로직 전담 (서버 전용). 시크릿 숫자 생성, 판정, 턴 관리 |
| `ANBGameStateBase` | 모든 클라이언트에게 동기화할 상태값 보관 (타이머, 현재 턴, 플레이어 목록) |
| `ANBPlayerController` | 각 플레이어의 입출력 처리 및 UI 관리 |
| `ANBPlayerState` | 플레이어별 상태 (이름, 현재 추측 횟수, 최대 추측 횟수) |

### 2. 핵심 게임 로직 구현
 
**시크릿 숫자 생성 (`GenerateSecretNumber`)**
- 1~9 숫자 배열에서 랜덤으로 3개를 추출
- `FDateTime::Now().GetTicks()`로 시드를 설정해 매 라운드 다른 숫자 보장
 
**입력값 검증 (`IsGuessNumberString`)**
- 3자리 여부, 숫자 여부, 0 포함 여부, 중복 여부를 순차적으로 검증
 
**결과 판정 (`JudgeResult`)**
- 자리와 값이 모두 같으면 Strike 카운트 증가
- 자리는 다르지만 `Contains()`로 숫자 존재 확인 시 Ball 카운트 증가
- 둘 다 0이면 `"OUT"` 반환, 나머지는 `"nSnB"` 형식으로 반환
 
**턴 처리 (`PassTurn`)**
- 현재 플레이어의 추측 횟수를 증가시키고 다음 플레이어로 인덱스 이동
- 마지막 플레이어 이후엔 인덱스를 0으로 순환
 
**승패 판정 (`JudgeGame`)**
- 3 Strike → 해당 플레이어 승리 메시지 브로드캐스트 후 `ResetGame()`
- Strike 미달 시 모든 플레이어의 `IsGuessCountMax()`를 확인해 전원 소진이면 무승부
 
---
 
### 3. 네트워크 동기화
 
서버에서 변경된 값을 클라이언트에 자동으로 반영하기 위해 `ReplicatedUsing`을 활용
 
```cpp
UPROPERTY(ReplicatedUsing = OnRep_RemainingTime)
int32 RemainingTime;
 
UPROPERTY(ReplicatedUsing = OnRep_CurrentPlayer)
int32 CurrentPlayer;
 
UPROPERTY(ReplicatedUsing = OnRep_PlayerList)
TArray<FString> PlayerList;
```
 
- 각 변수가 변경되면 `OnRep_*` 콜백이 클라이언트에서 자동 호출
- 콜백 내에서 로컬 `PlayerController`를 찾아 UI 업데이트 함수 호출
- 접속 알림은 `NetMulticast RPC`로 모든 클라이언트에 브로드캐스트
 
---
 
### 4. 플레이어 입출력 처리
 
**UI 동적 생성**
- `BeginPlay()`에서 5개의 위젯(채팅, 알림, 타이머, 플레이어 목록, 채팅 박스)을 동적으로 생성 및 뷰포트 추가
- 채팅 메시지는 `UTextBlock`을 동적으로 생성해 `UVerticalBox`에 추가
- 채팅 내역이 10개를 초과하면 가장 오래된 항목부터 자동 삭제
- 현재 턴 플레이어는 `SetColorAndOpacity(FLinearColor::Red)`로 빨간색 강조
